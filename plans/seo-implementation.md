# SEO and Runtime Optimization Implementation

## Goal

Ship the planned technical, SEO, GEO, and dead-code cleanup for `pseudobun.dev` while keeping the current two-locale structure and existing page count.

## Milestones

1. Fix type/build foundations and CI.
2. Refactor locale routing so `[lang]` owns the root layout without cookie-driven rendering.
3. Replace sitemap generation and rebuild localized metadata/schema.
4. Improve semantic content and project signals on the homepage and projects page.
5. Remove unused gallery/theme/query code and re-run verification.

## Progress

- Started repo audit and confirmed live sitemap, metadata, and cacheability issues.
- Validated that direct `next build` works and the app can be refactored incrementally.
- Replaced cookie-driven locale rendering with path-driven static routes under `src/app/[lang]`.
- Replaced `next-sitemap` with `src/app/sitemap.ts` and updated `robots.ts`.
- Added localized page metadata, canonical URLs, `hreflang`, JSON-LD, and semantic headings.
- Removed unused gallery/theme/query/ui code and added `pnpm typecheck`.
- Verified the new app shape with `pnpm lint`, `pnpm typecheck`, `pnpm build`, and local `curl` checks.

## Surprises & Discoveries

- The live `sitemap.xml` currently points to `robots.txt`.
- The current dynamic rendering is driven by `cookies()` in layouts rather than by actual data needs.
- `pnpm build` works in the current shell; the earlier symlink issue appears environment-related rather than a deterministic repo failure.
- A few unrelated public icon asset changes appeared in the worktree during implementation and were left untouched.

## Decision Log

- Keep `/` as an Accept-Language redirect and keep localized pages under `/sl/` and `/en/`.
- Remove unused gallery/query/theme code instead of preserving it for later.
- Replace `next-sitemap` with `src/app/sitemap.ts` to avoid stale generated XML artifacts.

## Outcomes & Retrospective

- The localized pages now build as static HTML, `/sitemap.xml` is generated by the App Router, and local verification shows route-correct canonical URLs, `hreflang`, JSON-LD, semantic headings, and cacheable localized responses.
